# 進化ポケモン名の正規化マッチング実装

## 背景

進化テーブルのポケモン名はテキストとして取得されるが、一覧ページの名前と完全一致しない可能性がある（全角/半角の違い、スペースの有無など）。名前→ID解決が失敗すると進化データが保存されない。

## 変更内容

### `internal/scraper/scraper.go`

- `normalizeName()` を追加: 名前を正規化する関数
  - 全角英数字（Ａ-Ｚ、ａ-ｚ、０-９）→ 半角に変換
  - 全角/半角スペース → 除去
  - 前後の空白 → トリム

- `resolveEvolutionName()` を追加: 3段階で名前→IDを解決
  1. 正規化済みnameMap（一覧ページのlinks由来）から照合
  2. DBから完全一致検索
  3. DBからLIKE部分一致検索（フォールバック）

- `Run()` を変更: 一覧ページの `links` から正規化済み `nameMap` を構築し、進化解決時に使用
