# FKエラー対策: 進化データの保存タイミング変更

## 背景

進化データの保存時にFOREIGN KEY制約エラーが発生。`nameMap` で名前→IDの解決は成功するが、実際のDBにまだそのポケモンがINSERTされていないためFK制約に違反していた。

## 変更内容

### `internal/scraper/scraper.go`

- `Run()` での進化データ保存を完全に2パス目に移行
  - 1パス目: 進化データを `allEvolutions` に収集するだけ（DBへの保存はしない）
  - 2パス目: 全386匹のINSERT完了後にまとめて進化データを解決・保存
- 1パス目での「試行→失敗→pending」の2段階処理を廃止し、シンプルな構成に変更
