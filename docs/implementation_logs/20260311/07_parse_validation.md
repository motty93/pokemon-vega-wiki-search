# パース結果のバリデーション追加

## 背景

パーサーの不具合でDBに不正なデータが保存されていたが、構造体が全て `string` / `int` のためコード上で異常を検知できなかった。DBに入れて初めて問題が発覚する状態だった。

## 変更内容

### `internal/model/pokemon.go`

- `validTypes` マップを追加: 有効なポケモンタイプ17種の定義
- `PokemonDetail.Validate()` メソッドを追加: パース後にデータの妥当性を検証
  - ID が 1〜386 の範囲内か
  - name が空でないか
  - type1 が空でないか、有効なタイプか
  - type2 が設定されている場合、有効なタイプか
  - ability1 が空でないか
  - 種族値が全て0でないか
  - image_url が空でないか、`https://` で始まるか
  - 複数のエラーをまとめて返す

### `internal/scraper/scraper.go`

- `ScrapePokemonPage()` の戻り前で `detail.Validate()` を呼び出し
- 旧来のデフォルト設定（`type1 == "" → ノーマル`）を削除し、バリデーションエラーとして検出するように変更

## 効果

パース失敗時にDBへの保存前にエラーが発生し、WARNINGログに出力される。不正なデータがDBに入ることを防止する。
