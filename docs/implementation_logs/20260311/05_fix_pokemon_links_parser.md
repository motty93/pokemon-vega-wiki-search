# 図鑑一覧ページのパーサー修正

## 背景

`FetchPokemonLinks()` がポケモンリンクを0件しか取得できなかった。WikiのHTML構造を調査した結果、想定と実際のテーブル構造が異なっていた。

- **想定**: 1行に `No.` と `名前` の2列
- **実際**: 1行に8匹分が横並び（No. + 名前 の2列 × 8 = 16列）
- ヘッダーもthではなくtdで `No.` `名前` × 8
- hrefは `//w.atwiki.jp/...` 形式（プロトコル省略）

## 変更内容

### `internal/scraper/scraper.go`

- `FetchPokemonLinks()` を全面書き換え
  - 各行のtdを2列ずつペア（No., 名前）で処理するように変更
  - IDの正規表現を `^(\d+)$` に変更（No.プレフィックスなしの数字のみ）
  - href のプロトコル補完を `baseURL +` から `"https:" +` に変更（`//` 始まりに対応）
  - 取得後にID順でソート（テーブルが横並びのため行順だと001→051→101→...になるため）
