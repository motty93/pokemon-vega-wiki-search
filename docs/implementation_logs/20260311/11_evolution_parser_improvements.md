# 進化パーサーの改善（分岐進化・進化しない対応）

## 背景

進化データ保存時に多数のWARNINGが発生。2つの原因があった。

1. 「進化しない」テキストがスキップされず、存在しないポケモン名として解決を試みていた
2. 分岐進化（1つのtdに複数のaタグがbr区切り）が名前結合されて1件として扱われていた
   - 例: `サーナイトエルレイド` が1つのポケモン名として解決失敗

## 変更内容

### `internal/scraper/scraper.go`

- `parseEvolutionRow()` を全面書き換え
  - 「進化しない」テキストをスキップ対象に追加
  - td内の複数aタグを個別に取得し、各ポケモン名と条件（括弧内テキスト）を分離
  - 分岐進化を個別のEvolutionDetailとして登録

### `migrations/001_init.sql`

- `evolution` テーブルに `UNIQUE(from_pokemon_id, to_pokemon_id)` 制約を追加（進化前後両方のページから同じ進化が登録される重複を防止）

## 結果

- 433件の進化データのうち、解決失敗は1件（シビシラス→シビビール、手動挿入で対応）
- 重複排除後201件のユニークな進化データ
- 分岐進化（キルリア→サーナイト/エルレイド等）も条件付きで正常に登録
