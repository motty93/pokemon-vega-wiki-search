# 検索結果表示領域の最小高さ確保

## 背景

検索結果が少ない場合、結果表示領域が短くスクロールしても結果が見づらかった。
ただし初期表示（検索前）ではスクロール領域を出したくないため、検索結果テンプレート側で制御する必要があった。

## 変更内容

### `templates/search_results.html`

- 結果あり時のラッパー `<div class="pt-20 pb-8">` に `min-h-screen` を追加
- 結果なし時（0件・未入力）のラッパー `<div class="text-center py-16">` に `min-h-screen` を追加

### 動作

- **検索前**: `#results` は空のため高さゼロ → ファーストビューでスクロール不要
- **検索後**: htmxで差し込まれるテンプレートに `min-h-screen` が含まれるため、結果が少なくても画面1枚分の領域が確保される
